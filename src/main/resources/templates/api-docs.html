API Overview

Welcome to the Training Center Management System API documentation. This RESTful API provides comprehensive access to manage students, trainers, courses, registrations, and grades.

Secure Authentication

Spring Security with BCrypt password hashing

JPA Persistence

Hibernate ORM with full relationship mapping

Role-Based Access

ADMIN, ETUDIANT, FORMATEUR roles

RESTful Design

Standard HTTP methods and status codes

Base URL
http://localhost:8080/api
Supported Content Types
  • application/json - JSON request/response
  • application/x-www-form-urlencoded - Form data

Authentication

Login Endpoint
POST /login

Authenticate user with credentials

Request Body
Parameter Type Required Description
username string Yes User login username
password string Yes User password (BCrypt encoded)
Demo Credentials
Username Password Role
mohamed admin123 ADMIN
saleh admin123 ETUDIANT
ali admin123 FORMATEUR
Response (Success)
{ "status": "success", "message": "Login successful", "user": { "id": 1, "username": "mohamed", "email": "mohamed@iit.com", "role": "ADMIN", "firstName": "Mohamed", "lastName": "Zayani" } }
Authentication Flow
  1. User submits credentials via login form
  2. Spring Security validates credentials against database
  3. Password is verified using BCrypt algorithm
  4. Session is created with user authorities
  5. User is redirected to dashboard

Error Codes

HTTP Status Code Description
200 OK Request successful
201 CREATED Resource created successfully
204 NO_CONTENT Resource deleted successfully
400 BAD_REQUEST Invalid request parameters
401 UNAUTHORIZED Authentication required
403 FORBIDDEN Insufficient permissions
404 NOT_FOUND Resource not found
409 CONFLICT Resource already exists (duplicate)
500 INTERNAL_SERVER_ERROR Server error occurred
Error Response Format
{ "timestamp": "2026-01-20T17:30:00", "status": 400, "error": "Bad Request", "message": "Invalid field value", "path": "/api/etudiants" }
